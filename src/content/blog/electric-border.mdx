Borders are boring. They sit there, static, doing the minimum. Give that border a moving pulse of light and suddenly your card feels powered on. That is what the Electric Border component does, and the whole thing is pure CSS under the hood.

## What we are building

A glowing light pulse travels continuously around the border of any container. The effect looks like electrical current moving through a wire, completing laps indefinitely. No canvas, no JavaScript animation loop. Just a rotating conic-gradient with a mask.

## Setting up

No dependencies beyond React. The component uses inline styles and a `<style>` tag for the keyframe animation, so it works anywhere.

```tsx
interface ElectricBorderProps {
  children: React.ReactNode;
  className?: string;
  duration?: number;
  color?: string;
  borderRadius?: string;
  borderWidth?: number;
}
```

## Building the component

The trick is combining a conic-gradient with CSS masking. Instead of painting a full border, we create a spinning gradient and then cut out the interior so only the edge shows.

```tsx
<div
  className='absolute inset-0'
  style={{
    borderRadius,
    padding: `${borderWidth}px`,
    background: `conic-gradient(from 0deg, transparent 0%, ${color} 10%, transparent 20%)`,
    WebkitMask: `linear-gradient(#fff 0 0) content-box, linear-gradient(#fff 0 0)`,
    WebkitMaskComposite: 'xor',
    maskComposite: 'exclude',
    animation: `electric-spin ${duration}s linear infinite`,
  }}
/>
```

The `conic-gradient` creates a sweep of color that is mostly transparent. Only a narrow band around the 10% mark is visible. By spinning this gradient with a CSS animation, that narrow band traces around the border.

The `WebkitMask` combination with `maskComposite: 'exclude'` is what turns the filled rectangle into just a border ring. It subtracts the content box from the full box, leaving only the padding area visible.

The keyframe itself is minimal:

```tsx
<style>{`
  @keyframes electric-spin {
    from { transform: rotate(0deg); }
    to { transform: rotate(360deg); }
  }
`}</style>
```

The full component wraps children in a `relative` container, places the animated border layer as an `absolute inset-0` overlay, and puts the children in another `relative` div so they sit on top:

```tsx
export default function ElectricBorder({
  children,
  className = '',
  duration = 3,
  color = '#ffffff',
  borderRadius = '12px',
  borderWidth = 2,
}: ElectricBorderProps) {
  return (
    <div className={`relative ${className}`} style={{ borderRadius }}>
      <div className='absolute inset-0 overflow-hidden' style={{ borderRadius }}>
        <div
          className='absolute inset-0'
          style={{
            borderRadius,
            padding: `${borderWidth}px`,
            background: `conic-gradient(from 0deg, transparent 0%, ${color} 10%, transparent 20%)`,
            WebkitMask: `linear-gradient(#fff 0 0) content-box, linear-gradient(#fff 0 0)`,
            WebkitMaskComposite: 'xor',
            maskComposite: 'exclude',
            animation: `electric-spin ${duration}s linear infinite`,
          }}
        />
      </div>
      <div className='relative' style={{ borderRadius }}>
        {children}
      </div>
      <style>{`
        @keyframes electric-spin {
          from { transform: rotate(0deg); }
          to { transform: rotate(360deg); }
        }
      `}</style>
    </div>
  );
}
```

## How to use it

Wrap any card or container:

```tsx
<ElectricBorder color="#4af" duration={2} borderWidth={2} borderRadius="16px">
  <div className="p-6 bg-zinc-900">
    <h2>Premium Plan</h2>
  </div>
</ElectricBorder>
```

## Key takeaways

- The conic-gradient plus CSS mask technique produces a true border ring with zero canvas or SVG overhead.
- `maskComposite: 'exclude'` is the key that cuts out the interior and leaves only the border strip visible.
- Duration between 1.5s and 3s hits the sweet spot. Faster feels jittery, slower loses the energy that makes it work.
