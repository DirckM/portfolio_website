Static backgrounds feel static. Softly undulating lines of color feel alive. The LineWaves component draws a set of sine curves on a canvas element and animates them frame by frame, giving any page a subtle pulse without a single animation library.

## What we are building

A canvas-based background that renders multiple horizontal sine waves. Each line oscillates at a slightly different phase and amplitude, so they never perfectly overlap. The result is a layered breathing effect that runs continuously at 60fps.

## Setting up

No external dependencies. The component uses the browser's `Canvas2D` API and `requestAnimationFrame`.

```tsx
import { useRef, useEffect } from 'react';
```

## Building the component

Start with the props interface and defaults:

```tsx
interface LineWavesProps {
  lineCount?: number;
  lineColor?: string;
  lineWidth?: number;
  amplitude?: number;
  frequency?: number;
  speed?: number;
  className?: string;
}
```

The canvas needs to respect the device pixel ratio so lines stay sharp on retina screens. Inside the `useEffect`, set this up before drawing anything:

```tsx
function resize() {
  if (!canvas) return;
  canvas.width = canvas.offsetWidth * window.devicePixelRatio;
  canvas.height = canvas.offsetHeight * window.devicePixelRatio;
  ctx!.scale(window.devicePixelRatio, window.devicePixelRatio);
}
```

The drawing function calculates evenly spaced base positions, then draws each wave with a sine offset that varies per line:

```tsx
function draw() {
  if (!canvas || !ctx) return;
  const w = canvas.offsetWidth;
  const h = canvas.offsetHeight;

  ctx.clearRect(0, 0, w, h);
  ctx.strokeStyle = lineColor;
  ctx.lineWidth = lineWidth;

  const spacing = h / (lineCount + 1);

  for (let i = 1; i <= lineCount; i++) {
    const baseY = spacing * i;
    ctx.beginPath();

    for (let x = 0; x <= w; x++) {
      const y =
        baseY +
        Math.sin(x * frequency + offsetRef.current + i * 0.8) *
          amplitude *
          (1 - Math.abs(i - lineCount / 2) / lineCount);
      if (x === 0) {
        ctx.moveTo(x, y);
      } else {
        ctx.lineTo(x, y);
      }
    }

    ctx.stroke();
  }

  offsetRef.current += speed;
  animationRef.current = requestAnimationFrame(draw);
}
```

The key part is `i * 0.8` added to the phase. Each line gets its own phase offset, so they oscillate out of sync. The amplitude modifier `(1 - Math.abs(i - lineCount / 2) / lineCount)` scales waves down toward the edges, making the center lines more dramatic.

The `offsetRef` accumulates each frame and moves the whole wave field horizontally over time. Advance it by `speed` per frame rather than using a timestamp for simplicity.

Kick off the loop and clean up:

```tsx
draw();

return () => {
  window.removeEventListener('resize', resize);
  cancelAnimationFrame(animationRef.current);
};
```

The full component renders a single canvas that fills its parent:

```tsx
return <canvas ref={canvasRef} className={`w-full h-full ${className}`} />;
```

## How to use it

Drop it inside any positioned container and set a background color on the parent:

```tsx
<div className="relative h-screen bg-neutral-950">
  <LineWaves
    lineCount={8}
    lineColor="#e5e5e5"
    amplitude={40}
    frequency={0.02}
    speed={0.02}
  />
  <div className="relative z-10">Your content here</div>
</div>
```

## Key takeaways

- The `devicePixelRatio` scale call is essential for crisp lines on high-DPI screens. Skip it and everything looks blurry.
- Varying the phase offset per line (`i * 0.8`) is what makes the waves look organic rather than synchronized.
- Keeping speed low (0.01 to 0.03) keeps the motion calm and readable behind content.
