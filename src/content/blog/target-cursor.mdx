The cursor is the one thing on screen that the user controls completely. Most sites ignore it and leave the default arrow. Swap that for a precision crosshair and you immediately change the feel of the entire interface.

## What it does

The native cursor is hidden and replaced with a custom crosshair that follows the mouse. The crosshair lags slightly behind using spring physics, giving the motion a satisfying weight. It can also expand or contract when hovering over interactive elements.

## How it works

Mouse position is tracked globally via a `mousemove` event listener. The custom crosshair element's position is updated using Framer Motion's `useSpring`, which interpolates toward the target position with configurable stiffness and damping values.

Hoverable targets get a data attribute that the cursor listens to. When the cursor enters one of those elements, it transitions to an expanded or outlined state.

## Key properties

- **size** - Default crosshair size in pixels (default: 32px)
- **color** - Crosshair stroke color
- **stiffness** - Spring stiffness for the follow lag (default: 400)
- **damping** - Spring damping (default: 28)
- **expandOnHover** - Whether to grow when over interactive elements
- **expandedSize** - Crosshair size in the hover state

## When to use it

Use this for portfolios, creative agencies, or any site where craft and attention to detail are part of the brand. It signals intentionality immediately.

Lower the stiffness value (200-300) if you want a floatier feel. Higher values (500+) make it feel snappy and precise, which suits a tactical or technical aesthetic.
